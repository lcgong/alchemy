### 如何选择 rebase 或 merge
1. 上游分支合并下游分支用 merge
1. 下游分支频繁同步（比如每天，分支间差异较小）上游分支改动用 rebase
1. 下游分支不频繁同步（比如一年，分支间差异极大）上游分支改动用 merge
1. 作为团队工作起点的主干分支，不要 rebase 其他分支
